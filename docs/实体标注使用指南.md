# NEXUS 实体标注系统使用指南

## 概述

NEXUS实体标注系统参考了shiji-kb项目的语法高亮功能，为历史文献提供丰富的实体标注和段落编号功能。

## 快速开始

### 1. 基本语法

所有实体标注使用统一的格式（推荐“标签前缀”写法）：

```
〖标签:内容〗
```

例如：
- `〖person:孔子〗` 或 `〖p:孔子〗` → 人名标注
- `〖place:长安〗` 或 `〖loc:长安〗` → 地名标注
- `〖official:丞相〗` → 官职标注

> 兼容旧写法：`〖@孔子〗`、`〖=长安〗`、`〖;丞相〗` 仍可用。

### 2. 消歧语法

当实体有简称或别名时，使用消歧语法：

```
〖标签:简称|全名〗
```

例如：
- `〖person:始皇|秦始皇〗` → 显示"始皇"，链接到"秦始皇"
- `〖person:武王|周武王〗` → 显示"武王"，指向"周武王"

### 3. 段落编号

在段落中使用方括号添加编号：

```
[1] 这是第一段
[2] 这是第二段
[2.1] 这是第二段的子段落
```

## 使用示例

### 示例1：历史叙事

```markdown
[1] 〖&周朝〗时期，〖@周公旦〗摄政当国，辅佐年幼的〖@成王〗。
他制定〖^礼乐制度〗，完善〖^分封制〗，使〖=西周〗进入鼎盛时期。

[2] 〖%公元前1046年〗，〖@武王|周武王〗率领〖$八百诸侯〗在〖=牧野〗
击败〖&商朝〗军队，建立了以〖=镐京〗为中心的统治体系。
```

### 示例2：文献引用

```markdown
〖{尚书〗记载：「王曰：『格尔众庶，悉听朕言。』」

〖{论语〗有云：子曰："学而时习之，不亦说乎？"
```

### 示例3：制度说明

```markdown
〖^科举制〗确立于〖&隋朝〗，完善于〖&唐朝〗。考生需通过〖;乡试〗、
〖;会试〗、〖;殿试〗三级考试，方能成为〖#进士〗。
```

## 高级功能

### 1. 引号内容高亮

系统会自动识别并高亮引号内容：
- 中文引号：`""` `''`
- 日式引号：`「」` `『』`

示例：
```markdown
古人云："民为邦本，本固邦宁。"
```

### 2. 鼠标悬停提示

所有实体标注都支持鼠标悬停显示类型信息：
- 简单格式：显示实体类型（如"人名"、"地名"）
- 消歧格式：显示完整名称（如"人名：秦始皇"）

### 3. 段落跳转

点击段落编号可以：
- 直接跳转到该段落
- URL会更新为该段落的锚点
- 便于分享和引用特定段落

## 编写建议

### 1. 标注原则

- **重要性优先**：优先标注对理解内容有关键作用的实体
- **保持一致**：同一实体在同一篇文章中使用相同的标注方式
- **避免过度**：不必对每个词都标注，影响阅读体验

### 2. 消歧使用场景

- 同名异人：`〖@文帝|汉文帝〗` vs `〖@文帝|魏文帝〗`
- 简称全称：`〖@始皇|秦始皇〗`
- 别名雅号：`〖@太白|李白〗`

### 3. 段落编号策略

- 重要段落：建议添加编号，便于引用
- 次要段落：可不编号，减少视觉干扰
- 分层次编号：使用多级编号组织复杂内容

## 技术细节

### 实现原理

- **Remark插件**：在Markdown解析阶段处理标注语法
- **CSS样式**：使用CSS变量支持主题切换
- **响应式设计**：适配各种屏幕尺寸

### 兼容性

- 完全兼容原有Markdown语法
- 支持与LaTeX数学公式混用
- 支持与其他Markdown插件协同工作

### 性能优化

- 编译时处理，不影响运行时性能
- 样式使用CSS变量，便于主题切换
- 轻量级实现，不增加额外依赖

## 常见问题

### Q1: 如何避免符号冲突？

推荐优先使用“标签前缀”语法（如 `〖person:孔子〗`、`〖legal:五刑〗`），不再依赖容易与 Markdown 混淆的符号。

- `〖` 和 `〗`：中文全角符号，可通过输入法或复制粘贴
- 旧符号语法仍兼容，但仅建议在已有历史内容中继续使用

### Q2: 实体标注可以嵌套吗？

不建议嵌套实体标注，可能导致显示异常。如果需要表达复杂关系，请使用消歧语法。

### Q3: 如何取消标注？

如果需要显示原始符号而不是标注，可以使用转义（目前不支持，建议避免冲突）。

### Q4: 段落编号从哪里开始？

段落编号可以随意开始，不需要连续。建议：
- 单篇文章：[1], [2], [3]...
- 分章节：[1.1], [1.2], [2.1]...
